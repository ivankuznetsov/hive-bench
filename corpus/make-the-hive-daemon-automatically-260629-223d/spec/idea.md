# make-the-hive-daemon-automatically-260629-223d

Make the Hive daemon automatically retry recoverable terminal error markers after the underlying dependency becomes healthy. We hit this during local setup on 2026-06-29: task 58 reached 4-execute and failed only because Codex auth was missing (codex exec returned 401 Missing bearer/basic auth). After Codex was fixed and CODEX_HOME=<REPO_ROOT> codex exec smoke-tested successfully, the daemon still left the task parked on an old implementer_failed marker requiring manual hive markers clear. Task 287 similarly stayed parked on claude_launch_failed after the live daemon was moved to the patched 0.3.2 Hive binary and hive doctor was green. Desired behavior: daemon should classify some error reasons as recoverable/transient, re-run lightweight health checks (e.g. hive doctor green, codex login status logged in, codex exec smoke test possible, Claude launcher ready detector fixed/current binary), and automatically clear/retry only when the marker matches a known recoverable reason and the health probe now passes. It must not auto-clear unknown/business-logic errors, repeated failing markers without a changed health signal, or markers where retry could discard user work. Add audit events/logging explaining why an auto-retry happened, retry limits/backoff to avoid loops, and tests covering Codex auth recovery and Claude launcher recovery.
